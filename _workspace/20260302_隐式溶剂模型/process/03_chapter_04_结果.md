# 第四章：运行与结果分析

## 4.1 运行计算

准备好所有输入文件（INPUT、STRU、KPT 及赝势文件）后，在计算目录下运行：

```bash
# 串行运行
abacus

# 并行运行（以 4 核为例）
OMP_NUM_THREADS=1 mpirun -np 4 abacus
```

计算完成后，输出文件位于 `OUT.H2/` 目录（对应 `suffix = H2`）。

## 4.2 在输出文件中找到溶剂化能量修正

打开 `OUT.H2/running_scf.log`，搜索 `E_sol_el` 和 `E_sol_cav`：

```bash
grep "E_sol" OUT.H2/running_scf.log
```

在每次 SCF 迭代中都会输出当前的溶剂化能量修正，最终收敛值出现在 SCF 完成后的汇总区域。

两个量的含义：

| 输出量 | 物理含义 | 通常符号 |
|--------|----------|----------|
| `E_sol_el` | 溶质-溶剂静电感应能 | 负值（稳定化） |
| `E_sol_cav` | 溶质在溶剂中形成空腔的能量代价 | 正值（不稳定化） |

对于 H₂ 分子这样的弱极性体系：
- `E_sol_el` 较小（H₂ 本身极性弱，溶剂的静电稳定化有限）
- `E_sol_cav` 同样较小（H₂ 体积小，占据的空腔小）

## 4.3 总能量与溶剂化能计算

SCF 完成后，`running_scf.log` 中同样输出总能量：

```bash
grep "FINAL_ETOT_IS" OUT.H2/running_scf.log
```

**溶剂化能的计算方法**：

溶剂化能 = E(imp_sol=1) − E(imp_sol=0)

具体操作：
1. 运行一次 `imp_sol = 1` 的计算（结构优化后），记录总能量 E_sol
2. 运行一次 `imp_sol = 0` 的计算（相同结构优化后），记录总能量 E_vac
3. 溶剂化能 = E_sol − E_vac

> 注意：严格来说，两次计算都应在**各自优化后的结构**下取能量，因为加入溶剂后结构可能发生细微弛豫。若仅做定性估计，也可用同一结构直接比较能量。

## 4.4 结果物理解读

**为什么 E_sol_el < 0？**

溶质的电荷（电子云 + 原子核）在连续介质中诱导极化电荷，诱导极化场又反过来作用于溶质，形成吸引性的静电相互作用，降低体系总能量。这是"溶剂化使分子稳定"的主要物理机制。

**为什么 E_sol_cav > 0？**

溶质"推开"溶剂分子，形成一个空腔，需要克服溶剂内部的分子间作用力（范德华力、氢键等）。这一过程耗费能量，对溶剂化自由能有正的贡献。

**溶剂化能的净结果**：

对于极性溶质（如含极性键的分子、带电体系），`|E_sol_el|` 往往远大于 `E_sol_cav`，净溶剂化能为负（溶剂化有利）。对于非极性溶质（如 H₂），两项都较小，溶剂化能接近零。
