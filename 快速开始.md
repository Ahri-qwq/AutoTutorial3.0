## 🚀 立即开始使用

### 方式1：简单主题测试

**对话示例：**
```
你：请按照CLAUDE.md生成一篇关于"ABACUS SCF自洽计算"的教程

Claude Code会：
1. 询问任务信息（是否有案例等）
2. 创建工作目录
3. 执行知识调研（调用retriever.py，学习参考文章风格）
4. 提供3个大纲方案供你选择
5. 撰写初稿
6. 三轮审查（内容、案例、风格）
7. 生成最终版本
```

### 方式2：案例驱动测试

**准备案例文件：**
- 将你的案例文件放到 `data/input/` 目录
- 支持格式：.docx、.md、.txt

**对话示例：**
```
你：
请按照CLAUDE.md生成一篇关于"ABACUS弹性常数计算"的教程，
案例文件是 data/input/elastic_case.docx，
请围绕案例展开

Claude Code会：
1. 询问确认（是否案例驱动）
2. 解析案例（调用case_parser.py）
3. 执行知识调研
4. 提供3个大纲方案（包含案例驱动型）
5. 撰写初稿（所有案例数据完整出现）
6. 三轮审查（特别是案例审查）
7. 生成最终版本
```

## 常见问题

### Q1: 我没有参考文章怎么办？
**A:** 可以：
1. 从data/runs/中找之前生成的好的教程
2. 从网上找ABACUS相关教程
3. 或者先跳过，直接测试（但风格可能不够好）

### Q2: 参考文章应该是什么样的？
**A:** 好的参考文章应该：
- 结构清晰（有明确的章节）
- 包含物理背景介绍
- 有详细的参数说明
- 有完整的代码示例
- 语言专业但易懂

### Q3: 工具测试失败怎么办？
**A:** 检查：
1. 知识库是否已初始化（`ls data/chroma_db/`）
2. config.yaml中是否配置了API Key
3. 依赖库是否已安装（`pip install -r requirements.txt`）

### Q4: 如何查看生成的教程？
**A:** 生成的教程在：
```
_workspace/YYYYMMDD_HHMMSS_主题/07_final.md
```

### Q5: 可以修改CLAUDE.md吗？
**A:** 可以！CLAUDE.md是灵活的指南，你可以根据需要调整：
- 修改流程步骤
- 调整写作规范
- 增加特殊要求

但核心原则（真实性、案例完整性、Think Aloud等）建议保留。
